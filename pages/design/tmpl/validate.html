Here are several links to mobile validation or page analysis tools related to mobile design. Use them to gage how well your website fits established mobile parameters or how well your pages conform to internet standards.

We have found many of them useful in identifying problems, and religiously use css and html5 validators in our code preparation. Validated code is always better. Analysis tools vary more widely in usefulness, but may be able to quickly highlight problem areas in you code or site design. Our code and site design is relatively complex, with module and content loading after initial page load. So our site process tends to score low on page load speeds and number of files. AngularJS and other dependencies are large files compressed or not, and modularity requires more seperate files. So as in life, everything is a compromise.
