OpenSiteMobile Development

Our original development goal was to create software for mobile websites. Later, we expanded that goal to that of creating adjustable display size, or responsive websites. And except for our 'msos.page' module, our code is relatively transparent by object oriented programming standards. It should be easy enough to "dumb down" as you like.

And our intent has always remained the same. We want to provide usable and flexible software that works on as many devices as possible. In that vane, we strive to incorporate as many of the best "open source" technologies and features available, into our software.

Development Features

  • Controlled Execution
  • Dependency Management
  • Localization
  • Modular Code
  • Responsive Web Design

Development Strategy

Our development strategy is not unlike what HTML5 BoilerPlate or BoilerPlateJS are trying to do to jump-start website development. As compared to HTML5 BoilerPlate or Mobile BoilerPlate, we have adopted a more holistic approach in that we are trying to integrate as many of the best mobile web technologies into a convenient "multi-page-content" development package. To that end, we have always used HTML5 BoilerPlate as a reference for base HTML5 page design best practices.

Thus our emphasis is more on the integration of basic structures needed to provide good website display on a variety of devices. With our modular code system, we use browser detection to load platform dependent code on demand. The same principles as prescribed by HTML5 BoilerPlate, but in a clean, multi-page environment. We aren't trying to re-invent the wheel, just make it roll more smoothly.
